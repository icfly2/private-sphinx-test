set_da_schedule(method='amp', **kwargs)[source]¶

Automatically schedule the da operation.

Schedule the DA operation using a predefined method. The follow methods are defined:

  • amp:
    • Initial scheduling method, according to which amp is reimbursed. Run when asset is 1 GBP in the money.
    • kwargs:
      • margin float: Margin by which the electricity price must exceed the gas prices, default = 1 GBP.
      • max_power float: Max power limit (%) of power to bid. Default is all of the assets power.
      • shift float: Number of hours in delay of price information.
        For initial scheduling use 24, for contract use 0. Default is 0.
      • contract bool: Not for public use. If this is a contract calculation or not,
        if contract, then the schedule is written to [‘schedule’,’contract’]. The Da profit share for DONG is also calculated there.
  • battery:
    • Charge or discharge a battery in a fixed time interval. Requires the kwargs start_time, end_time and direction.
      Always charges or discharges at the full asset power. Be careful, as this is an additive process and the schedule is altered, not replaced! Use method=lazy to initiate a clean slate.
    • kwargs:
      • start_time string: Time to start charging, eg: ‘05:00’, make sure both the start end end times are
        present in the index.
      • end_time string: End time of the interval in which to finish charging, eg: ‘06:00’.
      • direction string: (‘charge’/’discharge’), the direction of the flow in the battery.
  • lazy:
    • Do nothing. Asset is not scheduled on the day ahead market.
Parameters:
  • method (str) – Method with which to schedule.
  • **kwargs – Method dependant keyword arguments, see method description above for details.

Example

For a battery to charge in the morning and discharge in the afternoon:

analysis.schedules.set_da_schedule(method=’lazy’) analysis.schedules.set_da_schedule(method=’battery’,start_time = ‘05:00’, end_time = ‘06:00’, direction= ‘charge’) analysis.schedules.set_da_schedule(method=’battery’,start_time = ‘15:00’, end_time = ‘16:00’, direction= ‘discharge’)
set_imba_schedule(method='realistic', **kwargs)[source]¶

Automatically schedule the imbalance operation.

The following scheduling methods are pre defined:

  • perfect:
    • Trigger imbalance when ever it is profitable to do so.
    • kwargs:
      • margin (float): Set the minimum profit what must be made in a half hour to trigger imbalance. Default is 0.
  • realistic:
    • Trigger imbalance when the worst of the preceeding imbalance cost periods, indicates the current period may be profitable.
    • kwargs:
      • margin (float): Additional profit requirment. Default is 0, can be negative.
      • window (int): Number of past half hour windows to consider. Default is 2.
      • stochastic (bool/float): Default is False, otherwise enter fraction of timesteps to scramble. Simulates wrong calls.
-lazy:
  • Don’t participate in the imbalance market.
Parameters:
  • method (str) – Method with which to schedule.
  • **kwargs – Method dependant keyword arguments, see method description above for details.
set_schedule(time, schedule)[source]¶

Set the schedule for a time (DA, WD, Imbalance)

The index must be in time step format. The schedule in MW of power operation. A schedule tha serves as an input to another operation is written to both dataframes. Should be used instead of the predefined methods if a custom schedule was computed.

Parameters:
  • time (str) – Name of the market to schedule (DA,WD,Imbalance)
  • schedule (pd.Series) – Series of power settings. Must have the same time index as the original schedule data frame. Hourly data for DA, HH data for wd and imbalance.

set_wd_schedule(method='lazy', **kwargs)[source]¶

Simulate the within day trade using a predefiend method

The following trading methods are defined:

  • lazy:
    • Do not trade on the intraday
    • kwargs: None
  • greedy:
    • Take all profitable trades
    • kwargs:
      • capacity_charge (float): Charge per MWh with which to penalise deviations from the DA schedule.
      • margin (float): Pounds in total of profit that must be made per trade.
      • max_power_share (float): Proportion of max asset power to use for scheduling Default is 1, must be between 0 and 1.
      • max_trades (int): Maximum number of permissable trades per half hour trading block, default is 99.
      • trade_delay (pd.Timedelta): Time for which a trade must at least be available in the order book for it to be used in trading. 15 second granularity.
  • perfect:
    • Assuiming perfect market knowledge, select the best offers.
    • NOT IMPLIMENTED YET
Parameters:
  • method (str) – Method with which to schedule.
  • **kwargs – Method dependant keyword arguments, see method description above for details
